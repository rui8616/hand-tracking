# 产品方向调研报告

> 调研日期：2026-05-21  
> 状态：已选定方向 — 儿童近视防控 App，先做「用眼姿势监测」轻量模块切入

---

## 0. 起点 — 手势 demo（本仓库当前状态）

基于 MediaPipe Hand Landmarker + Three.js 的纯浏览器手势 demo：
- 实时手部 21 关键点追踪
- 「握拳 + 伸食指」蓄力，粒子从远处汇聚到指尖
- 满级后甩食指发射"洞洞波"，速度按角速度映射

技术验证：3D 骨骼可视化、手势状态机、粒子系统、能量球特效均可行。
问题：作为玩具完成，**作为产品需要找真实需求场景**。

---

## 1. 关键筛选准则：手势 vs 触屏的独占场景

经过多轮否决（演示控制器、视频会议、节奏游戏、空中签名都被触屏可替代）后，提炼出**手势真正不可被替代的四类场景**：

1. **手够不着屏幕** — 远屏、投影、舞台、大屏装置
2. **手在做别的事** — 油手做饭、抱孩子、握画笔、健身、开车
3. **触摸是禁忌** — 医疗无菌、戴手套、公共展品卫生顾虑
4. **触屏做不到的维度** — 3D 空间、双手协作、隔空操作

**不符合上述任意一条的方向都是 demo，不是产品。**

## 2. 已否决的方向及原因

| 方向 | 否决理由 |
|---|---|
| 演示控制器 Chrome Extension | 触屏 + 快捷键完全够 |
| 视频会议增强 | 鼠标键盘已经够，巨头会内置 |
| TikTok 风格滤镜 | 手势是表演不是必需，巨头主导 |
| 空中签名 / 写字 | 触屏 + Apple Pencil 完爆，MediaPipe 精度也不够 |
| 节奏体感游戏 | 触屏音游满足度更高，纯视觉手势缺触感反馈 |
| 演讲 / 商务礼仪训练 | 市场太垂直 |
| 厨房助手 | 油手是真实痛点，但菜谱 App 用户付费意愿低 |
| 品牌互动装置 / VJ | 是服务业不是产品，且单子定制工作量重 |
| 手部康复严肃医疗 | 需要医疗合规 + 临床验证 + 医生顾问，门槛太高 |
| 自闭症 / 感统训练 | 需要医疗顾问，B2C/B2B 销售都难做 |
| 幼儿园 / 早教中心 B 端 SaaS | 需要打销售，回款慢，单子小 |

## 3. 关键竞品调研（web 验证）

### 3.1 Osmo（最像我们方向的对标）— 死了

- 2014 年创立，摄像头识别实物 + iPad 配套早教游戏
- **2019 年印度 Byju's 以 $120M 收购**
- **2024 年 Byju's 进入破产程序**
- **2025 年 12 月 Osmo 被小团队从破产中买回**

启示：「儿童 + 摄像头识别 + 教育」过去 10 年里跑过最有钱的实验，**没跑通**。

### 3.2 Active Arcade — 找不到付费模式

- 手机摄像头全身体感游戏（4-9 岁孩子爱玩）
- 评测显示孩子粘性强
- **完全免费、无广告、无订阅、无内购**
- 母公司 Nex 转向卖 **Nex Playground 硬件**变现

启示：**纯软件体感学习 App 已被验证打不通**，能赚钱的还是硬件（任天堂、Nex、旧 Osmo）。

### 3.3 中国市场红利与天花板

- 「双减」后 **70.4% 用户增加教育智能硬件支出**（真实红利）
- 早教市场规模千亿级
- 但典型订阅价 **28 元/月、168 元/年** — 远低于直觉估的 600-1200 元/年
- 关键词「互动、陪伴、监督」与体感产品天然契合

启示：政策有红利，但 ARPU 低，单位经济压力大。

## 4. 选定方向：**儿童近视防控 App**

### 4.1 为什么这个方向过筛

| 维度 | 评估 |
|---|---|
| 手势 vs 触屏独占性 | ✅ 远眺训练强制远离屏幕，触屏天然不可行 |
| 付费意愿 | ✅ 「近视不可逆」是真实恐惧，高于早教 |
| 政策环境 | ✅ 不属于双减学科类，教育部明确支持近视防控 |
| 团队规模 | ✅ 个人/小团队可起步，不需要医疗顾问 |
| 既有竞品 | ✅ 目目伴、飞利浦 EyeRefuse 等都是硬件，软件入口空缺 |
| 客厅硬件普及 | ✅ 中国家用投影仪销量爆发，60+ 寸电视普及 |

### 4.2 核心交互模型

```
   ┌─────────────┐
   │  投影/电视   │  ← 大屏显示内容
   └─────────────┘
          │
          ↓ 4-5 米外
   ┌─────────────┐
   │  孩子站这里  │  ← 远眺距离，自然放松睫状肌
   └─────────────┘
          ↓ 伸手"指"目标
   ┌─────────────┐
   │  摄像头      │  ← USB / 旧手机 PWA / 笔记本前摄
   └─────────────┘
```

**核心命题**：屏幕本身是问题，App 必须**让屏幕变成「远眺工具」而不是「凑近看的玩具」**。

### 4.3 功能模块（5 大块）

1. **远眺训练游戏（核心）** — 屏幕显示移动目标，孩子在远处用手势"打"中。强制睫状肌切换聚焦深度。
2. **眼球运动训练** — 屏幕光点引导眼球做轨迹运动，Face Landmarker 验证眼球是否真的动。
3. **用眼姿势监测（被动）** — 孩子写作业 / 用 iPad 时后台运行：脸距、低头、揉眼、未眨眼、光线不足提醒。
4. **20-20-20 番茄钟** — 每 20 分钟强制远眺挑战，不完成则锁定其他 App。
5. **家长视图 + 视力档案** — 数据报告、PDF 导出、家庭自测、长期趋势。

### 4.4 6 周 MVP 范围（完整产品）

| Week | 内容 |
|---|---|
| 1-2 | 远距离手势识别精度验证（4 米外 MediaPipe）+ 投屏方案打通 |
| 3 | 1 个游戏主题（海底/太空二选一）×5 关 |
| 3-4 | 用眼姿势监测独立模块（脸距、低头、揉眼）|
| 4-5 | 20-20-20 番茄钟 + 家长报告 |
| 5 | 注册 / 儿童档案 / 投屏新手引导 |
| 6 | 内测 + 留存数据采集 |

砍掉到 v2：眼球运动训练、视力自测、内容订阅商店、多子女、教师端。

### 4.5 关键产品决策

1. **硬件假设最小化** — 用户已有的硬件最重要。摄像头方案：USB / 笔记本前摄 / 用旧手机扫码当无线摄像头（PWA）。
2. **不能宣称「治近视」** — 中国《广告法》严管医疗效果宣传。定位为「用眼习惯管理 + 视力健康陪伴」，靠"近视防控"政策词。
3. **数据本地处理** — 摄像头数据 100% 本地（MediaPipe 完全可以），不上云。这是相对传统教育 App 的真正护城河（家长隐私焦虑）。
4. **两面博弈**：家长付费看数据、孩子使用看游戏。两边都要平衡。
5. **客单价**：免费层（姿势监测 + 番茄钟） + 39 元/月 / 299 元/年（远眺游戏 + 内容订阅） + 49 元/月家庭版（多孩 + 档案）。

### 4.6 风险地图

| 风险 | 缓解 |
|---|---|
| 远距离手势精度（4 米外）不够 | **Week 1-2 必须实测**，跑不通整个方向就要换 |
| 小孩注意力不够 15 分钟 | 从 7-12 岁切入，避开低龄 |
| 客厅深度 < 3 米 | 支持"小客厅模式"，更近距离 + 更小目标 |
| 「治近视」效果证明不了 | 营销定位是"管理用眼习惯 + 让孩子动起来"，不打疗效牌 |
| 学校渠道复杂 | 先 C 端家庭跑通，后期再做学校 B 端 |

## 6. 重大方向调整（2026-05-21）

### 6.1 「完整近视防控 App」逻辑被推翻

§ 4 设计的「远眺训练大屏 + 4-5 米手势识别」**产品定义已废弃**。

被推翻的核心原因：
- **「让屏幕变大变远 = 解决近视」与医学共识冲突** — 户外光照才是关键，App 让小孩在客厅看 4 米屏幕，依然是「室内 + 屏幕」组合，护眼效果比户外活动差一个量级
- **家长真要防近视，第一反应是带孩子去户外，不是买另一个屏幕产品**
- **「为什么不直接去公园？」这个反问 App 答不上来** — 这意味着完整产品建立在一个站不住的需求假设上

### 6.2 留下来的真东西：用眼姿势监测

姿势监测的需求**不依赖「治近视」叙事**而独立成立：
- 孩子用屏幕是**不可避免的**（网课、电子作业、动画片）
- 屏幕时间内的「脸距 / 眨眼 / 揉眼 / 低头 / 光线」是真痛点
- 家长诉求是「**我尽力管理好了**」的心理交代，不是疗效

**新独立定位：屏幕时间健康助手**（Screen Time Health Assistant）
**不是**：近视防控产品、家长监控工具、医疗设备
**是**：焦虑安抚 + 行为可视化 + 隐私优先

### 6.3 终局规模调整 — 「小而美」路线

明确按 **B 类型（小品牌精品）** 路线规划：

| 维度 | 原预期 | 真实预期 |
|---|---|---|
| 核心叙事 | 近视防控（疗效暗示） | 屏幕时间健康助手（焦虑管理） |
| 定价能力 | 39-49 元/月 | 9.9-19.9 元/月 |
| 客单 LTV（年） | 300-500 元 | 100-200 元 |
| 用户规模 | 想百万级 | 几千付费用户的精品 |
| 终局 | 独角兽（不太可能） | 小而美 SaaS、被收购、或多产品矩阵第一个 |
| 团队规模 | 拿融资创业 | 1-3 人小团队、不投放、不烧钱 |
| 增长策略 | 投放 + 拉新 | 内容（小红书 / 公众号 / 知乎）+ 口碑 |

### 6.4 明确的「不做」清单

为了守住小而美定位，**永远不做**：
- ❌ 远眺训练大屏产品（被验证为伪需求）
- ❌ 学校 B 端 SaaS（不是这个团队的活）
- ❌ 「治近视」宣传（中国《广告法》合规风险）
- ❌ 拿融资扩张（破坏小而美定位）
- ❌ 投放买量（CAC > LTV，必死）
- ❌ Native App 抢跑（PWA 验证后再考虑）

### 6.5 决策红线（继承自原 MVP § 9）

3 周开发完成后，**D7 留存 < 30% 或付费意愿 < 5% 就放弃这个方向**，不被沉没成本绑架。

## 7. 最终方向切换（2026-05-22）— 办公族颈椎拉伸 App

### 7.1 「屏幕时间健康助手」被进一步推翻

§ 6 选定的「屏幕时间健康助手」（儿童版）在 Apple 生态约束下走不通：

1. **iOS 后台不能访问摄像头**（平台硬限制，没有变通）
2. **iOS 17+ 已原生集成「屏幕距离检测」**（系统级 Face ID 自动检测，挤占核心卖点）
3. **否决「双设备模式」后**，唯一形态退化为「护眼番茄钟」
4. **「护眼番茄钟」过于简单**，监测时长大幅缩水、价值密度不足

整个儿童方向至此封死。

### 7.2 复盘失败模式

回看前 6 节所有方向（演示控制器→早教→近视防控→自闭症→幼儿园 B 端→儿童屏幕健康），都被否决，共同失败模式是：

> **我一直在给手势找「软用例」**（炫技、新鲜感、点缀），而手势识别真正不可替代的场景是「**身体本身就是输入**」 — 用户在镜头前做身体动作，App 识别评估这些动作的**对错 / 次数 / 质量**。

之前所有方向手势都只是「输入方式之一」，触屏能替代或巨头能覆盖。

### 7.3 收紧后的过滤准则

**手势 / 姿势识别真正不可替代的场景**：

- ✅ 用户必须在镜头前**持续可见**（不是偶尔挥一下）
- ✅ 双手在做别的事（举铃、抓杆、握画笔、做动作） — 触屏物理上做不到
- ✅ **动作本身就是核心价值** — 不是控制别的，是被识别评估的对象

这一收紧之后，留下的母市场是：**「在镜头前做身体动作，App 识别评估」**。

### 7.4 已商业验证的同赛道产品

| 产品 | 商业模式 | 状态 |
|---|---|---|
| Onyx（AI 健身教练） | $14.99/月 | 跑通 |
| Apple Fitness+ | $9.99/月，50M+ 订阅 | **没做动作识别**，留有空间 |
| Tonal / Mirror / FightCamp | 硬件 + 订阅 | 跑通但需硬件 |
| Tempo Move | 硬件 + AI 姿势 | 跑通 |
| Yousician（音乐版） | $19.99/月 | 跑通 |

**关键洞察**：Apple Fitness+ 自身没做动作识别，第三方在 iOS 生态有真空。

### 7.5 最终选定：办公族颈椎拉伸 App

按"用户付费意愿 × 真实痛点 × 小而美适配"三轴评估，从健身赛道里收敛到具体场景：

**目标用户**：30-45 岁互联网/办公室白领（首推程序员、设计师）

**为什么是这个细分**：

1. **痛点最近**：自己就是程序员，知道这个痛 — 「为自己做产品」是创业最强起点
2. **用户最容易触达**：程序员社区（V2EX、即刻、X、掘金）天然在线
3. **付费意愿最确定**：高 ARPU 群体 + 健康焦虑 + 「每月 30 块比看一次理疗便宜」
4. **没有医疗合规风险**：不承诺治病，定位"办公室健康习惯"
5. **小而美完美适配**：1 人 3 个月可做出 V1 验证

### 7.6 决策最终明确

| 项 | 决定 |
|---|---|
| 赛道 | 成人健身 / 养生（彻底放弃儿童市场） |
| 细分 | 办公族颈椎 / 腰椎 / 手腕拉伸 |
| 主用户 | 程序员 / 设计师 / 办公族 |
| 平台 | iOS / iPadOS Native（彻底放弃浏览器版作为正式产品） |
| 不做 | macOS、儿童、医疗、硬件、B 端、双设备 |
| 营销 | 程序员社区内容营销（V2EX、即刻、X、ProductHunt） |

详细规划见 [neck-stretch-app-mvp.md](./neck-stretch-app-mvp.md)。

### 7.7 这一轮的反思

本次产品方向调研共经历 **6 次大方向调整 + 数次否决**，最终从 demo（手势洞洞波）走到清晰的产品定义。

留给未来的几条经验：

1. **「列方向」不是产品决策** — 每次罗列 8 个方向都是没想清楚的表现，应该深入挖第一个方向直到它垮掉再换
2. **过滤准则比方向重要** — 「手势 vs 触屏独占场景」收紧到「身体本身是输入」才真正定向
3. **承认伪需求** — 完整近视防控被推翻不是产品问题，是需求假设的错。早承认早换
4. **小而美是约束不是目标** — 它逼着你放弃大叙事 / 大融资 / 大团队，反而能聚焦
5. **为自己做产品最稳** — 用户群里有你自己 → 校准最低成本
