# 办公族颈椎拉伸 App — MVP 规划

> 文档版本：v1（2026-05-22）  
> 决策背景：[product-research.md § 7](./product-research.md#7-最终方向切换2026-05-22--办公族颈椎拉伸-app)  
> 定位：iOS / iPadOS Native 应用，目标 1-3 人小团队跑出年流水百万级别的现金流业务  
> 工程代号：**StretchPal**（正式品牌名发布前用）

---

## 1. 产品定位

**一句话**：办公族（程序员、设计师、白领）每天 10-15 分钟跟着 App 做颈椎/肩部/腰部拉伸，**摄像头实时判定动作是否做到位**，给出量化的健康习惯报告。

**这是什么**：
- 一个比 YouTube 跟练 + 更智能的「办公健康陪伴」
- 摄像头看你的动作角度、停留时长、是否对称
- 让「我练了但不知道动作对不对」变成「App 告诉我每个动作做对了几分」
- 100% 本地处理摄像头数据（Apple 用户极看重隐私）

**这不是什么**：
- ❌ 不是医疗设备、不诊断颈椎病、不治疗椎间盘突出
- ❌ 不是健身房替代品（不做力量训练）
- ❌ 不是 Keep / Apple Fitness+ 那种内容型大平台
- ❌ 不是康复医疗 App（避开合规）
- ❌ 不是「久坐提醒」工具（那是辅助功能，不是核心）

**核心叙事**：
> 「你知道自己该练，但不去健身房、不会请理疗师、看 YouTube 视频又不知道动作对不对。**让摄像头当你的私人物理治疗陪伴**。」

**为什么用户会付费**：
- **痛点真实**：90% 程序员、设计师有颈椎问题，自己心里清楚
- **替代方案贵**：一次理疗 200-500 元；私教 300-800 元/次
- **价值锚点**：「19 元/月 = 一次理疗的 5%，但天天能用」
- **解决「不知道对不对」**：跟 YouTube 跟练最大的痛点

## 3. 核心功能（MVP 范围）

### 3.1 拉伸训练库

**6 个模块 × 5-8 个动作 = 约 30-40 个动作**：

| 模块 | 关键动作 | 关注点 |
|---|---|---|
| 颈部 | 前屈、后伸、侧倾、旋转、米字操 | 角度、停留 |
| 肩部 | 耸肩、绕肩、肩胛骨内收、肩部前伸 | 对称性、范围 |
| 上背 | 猫牛式、胸椎旋转、墙壁俯卧撑 | 脊柱曲度 |
| 下背 | 桥式、转脊、腰部前屈、髋部拉伸 | 骨盆稳定 |
| 腰部 | 婴儿式、麻花式、坐姿前屈 | 柔韧性 |
| 手腕 | 手腕拉伸、手指张合、前臂滚 | 完成度 |

每个动作有：
- 30-60 秒示范视频（实拍，请专业人士录制）
- 标准动作的关键点说明（文本 + 图示）
- 摄像头判定阈值（如颈部前屈 ≥ 40°）
- 错误识别（如"代偿动作"提示）

### 3.2 摄像头实时判定（核心差异化）

训练界面：
- 上半部分：示范视频或动画
- 下半部分：摄像头小窗（用户自己的画面）
- 摄像头实时画**关键关节点 + 角度数字 + 标准/不标准颜色**
- 完成判定：达到目标角度 + 保持 N 秒 = ✅
- 不标准时给具体反馈："肩膀偏斜了，左肩抬高一点"

### 3.3 每日训练计划

- 默认 3 个时段：早晨（5min）/ 午间（10min）/ 傍晚（5min）
- 用户可调整、可选其中 1-2 个
- 计划智能轮换不同部位，避免每天重复
- 用 iOS 通知温和提醒（不打扰、可关闭）

### 3.4 量化进度报告

- **训练记录**：每天完成几个动作、每个动作的"标准度评分"
- **关节灵活度趋势**：颈椎旋转角度本周比上周多了 5°
- **连续打卡**：解锁勋章（不强推社交分享）
- **周报**：每周日推送一次，文字+图表

### 3.5 隐私优先（卖点）

- 摄像头数据**100% 本地处理**（Vision framework + Neural Engine）
- 不录像、不上传、不分析用户画面
- 训练数据用 iCloud 同步（用户自己的 iCloud，不经我们服务器）
- 在 App 启动页 + 设置页常驻"无云端、无服务器"承诺

## 4. 姿势判定算法

### 4.1 技术基础

iOS 提供两个相关 API：

| API | 能力 | 适用 |
|---|---|---|
| **Vision framework `VNDetectHumanBodyPoseRequest`** | 17 点身体姿势（COCO 格式） | 主用 |
| **ARKit Body Tracking** | 91 点全身 3D（仅 iPhone X+ + iPad Pro） | v2 备选 |
| **Vision `VNDetectFaceLandmarksRequest`** | 76 点脸部（颈椎判定需要） | 颈部模块 |

主选 Vision + Pose，跨设备覆盖最广。

### 4.2 关键指标计算

**颈椎前屈角度**：
```
取 face landmark (鼻尖 / 下巴) + body pose (左右肩中点)
颈椎角度 = 头部矢状面与躯干矢状面的夹角
```

**肩部对称性**：
```
左肩 y - 右肩 y → 偏差度数
正常 < 3°，警告 > 5°
```

**躯干前屈**：
```
取 hip (左右髋中点) 与 shoulder (左右肩中点)
前屈角 = 躯干向量与垂直向量的夹角
```

**动作停留时间**：
```
关键关节角度连续在目标范围 ±5° 内 ≥ N 秒 = 完成
```

### 4.3 算法挑战

1. **侧向动作**：很多拉伸是侧面做的，Vision 的 2D pose 在侧面遮挡时精度差 → 给用户引导面对 / 侧对摄像头
2. **手腕动作太精细**：身体 pose 拿不到手指 → 用 MediaPipe Hand 补足（v1.5）
3. **代偿动作**：用户为了到位会用错误肌群 → 检测姿态的"组合特征"（如颈部前屈时肩部是否上提）
4. **个体差异**：柔韧性差异大 → 用前 3 天采集用户的"基线"，后续做相对评分

## 7. 数据模型

```swift
struct Exercise: Codable, Identifiable {
    let id: String          // "neck-flexion"
    let module: Module      // .neck / .shoulder / .upperBack ...
    let title: String       // "颈部前屈"
    let durationSec: Int    // 30
    let demoVideoURL: URL
    let keyJoints: [Joint]  // 算法关注哪些关节
    let targetAngle: Range<Double>?  // 40°-60°
    let symmetryRequired: Bool
    let holdSec: Int        // 3
}

struct TrainingSession: Codable, Identifiable {
    let id: UUID
    let startedAt: Date
    let endedAt: Date?
    let plan: [Exercise]
    let results: [ExerciseResult]
}

struct ExerciseResult: Codable {
    let exerciseId: String
    let score: Double       // 0-100 (标准度)
    let maxAngle: Double
    let holdAchieved: Bool
    let feedback: [String]  // 实时给过的提示
}

struct UserProfile: Codable {
    let createdAt: Date
    let ageRange: AgeRange  // .twenties / .thirties / ...
    let focus: Set<Module>
    let workSittingHours: Int
    let baseline: [Module: BaselineMeasurement]
}
```

存储：Core Data，iCloud 自动同步。

## 9. 定价（小而美定位）

| 等级 | 价格 | 功能 |
|---|---|---|
| 免费 | 0 元 | 8 个核心动作 + 不限次数训练 + 7 天历史 |
| Pro 月费 | ¥19 | 全部 30+ 动作 + 全部历史 + 智能计划 + 周报 |
| Pro 年费 | ¥158 | 同上 + 节省 30% |
| 终身买断 | ¥298（一次性）| 同上 + 终身使用 |

**为什么这个价格**：
- 19 元/月 ≈ 一杯咖啡，是程序员"无脑订阅"价位
- 158 元/年是 Keep 会员价位，对标心理价
- 终身买断 298 元给"反订阅"用户（程序员普遍反感订阅）
- Apple 抽 15-30%，实收约 13-16/月、110/年、208/终身

**单位经济测算**：

| 阶段 | 目标 | 月流水（扣 Apple）| 备注 |
|---|---|---|---|
| 6 个月内 | 50 付费 | ~700 元 | PMF 验证 |
| 12 个月 | 500 付费 | ~7000 元 | 第一个续费周期 |
| 24 个月 | 2500 付费 | ~35000 元 | 1 人可全职 |
| 36 个月 | 8000 付费 | ~110000 元 | 1-2 人小团队稳定 |

**8000 付费 = 这个产品的"小而美"上限假设**。超过这个量级要么进化（手腕/腰部专项 App 矩阵），要么承认到顶。

## 11. 产品演进路径

```
v1（本规划）— 颈椎 + 肩部基础 30 动作
              6 周 MVP，6 个月跑 PMF
                ↓ 触发条件：500 付费 + 续费率 > 50%
v1.5        — 加腰部 + 手腕 + MediaPipe Hand
              新增 15 动作
                ↓

              + WidgetKit 小组件（今日打卡）
                ↓

              复用引擎，包装不同场景
                ↓
v3 (探索)   — Apple Watch 配套：手腕震动提醒站起来
              + watchOS 简单训练
```

### 永远不做（划红线）

| 不做 | 原因 |
|---|---|
| 力量训练 / 健身房动作 | 偏离办公健康定位 |

| Android 版 | 散群体多、Apple 用户付费意愿稳 |
| Web 版正式产品 | 仅保留为 PoC / 落地页 demo |
| 硬件 / 智能镜子 | 资金需求大、违背小而美 |
| 拿融资 | 破坏小而美定位 |
| 抖音 / 微信运动模式 | 不需要社交 |
| 治病 / 康复宣传 | 合规风险 |
| B 端公司福利团购 | 销售复杂、回款慢 |

## 12. 几个开放问题

1. **示范视频谁来拍？** — 首批可找拉伸 / 瑜伽教练录 30 个动作，预算 5000-10000 元。或者用动画（成本低但效果差）。**推荐找真人**。
2. **个体差异如何处理？** — 前 3 天采集"基线"，之后做相对评分。柔韧性差的人能看到自己的进步，更有动力。
3. **iCloud 同步够吗？** — 单设备用户够，多设备（iPhone + iPad）也够。如果要做 Web 端 / 跨平台，需要自建后端，**v1 不做**。
4. **数据如何"打动"用户？** — 周报必须有 1 个"哇"点：「你的颈椎旋转比上周多了 5°」「连续 14 天打卡，超过了 87% 的用户」。
5. **付费墙怎么设计？** — 免费 8 个动作，但训练页**功能完整**（不限次数 + 摄像头判定）。Pro 解锁的是"更多内容 + 计划 + 报告"。

## 13. 立即可做的下一步

技术验证（**应在动手大量开发前完成，3-5 天**）：

1. **iOS 项目骨架** — Swift + SwiftUI + Vision framework hello world
2. **Vision Pose PoC** — 摄像头看自己做颈部前屈，console 打印关键关节角度
3. **角度算法验证** — 在不同光照、不同距离、不同坐姿下测试稳定性
4. **示范视频草拍** — 自己拍一个动作的示范视频，验证训练页 UX 流程

如果这四步通过，再按 6 周计划展开。

如果 Vision Pose 在你的真实场景（坐姿 + 普通光照）下精度不够，要考虑：
- 用 MediaPipe Pose 替代（开源、跨平台、但要自己包装）
- 或自训一个轻量模型（Core ML），针对"上半身办公场景"

## 14. 决策红线（写给未来的自己）

**永远不要被沉没成本绑架。**

6 周 MVP 完成 + 6 个月内测后看以下硬指标：

| 指标 | 红线 | 不达标的行动 |
|---|---|---|
| D7 留存 | < 30% | 砍掉、重新设计核心训练流程 |
| 付费转化率 | < 3% | 重新定价 / 重做付费墙 |
| 年付续费率 | < 50% | 重做核心价值（数据报告 / 进步可视化）|
| 月被动新增（无投放） | < 30 | 内容策略失败、重做营销 |
| 用户「再次打开 App」次数 | 平均 < 3 次/周 | 习惯没养成、需要更强提醒 |

**任一红线撞墙就停下来重新评估**，不要硬着头皮做 v1.5。

## 15. 给自己的一条提醒

这个产品的真正价值不是"动作识别多准"，是**让用户每天 10 分钟动起来 + 看到自己在变好**。

- 算法做到 80 分就够，剩下 20 分留给 UX / 内容 / 报告
- 不要陷入"无尽地调阈值"的工程师陷阱
- **能看到进步的用户会续费，看不到的不会**

营销文案直接抄这条：

> 「你不需要更好的颈椎按摩仪，你需要每天 10 分钟真的练。这个 App 让你看见自己的进步。」
